Streamlining the use of Twitch EventSub

Easily subscribe to Twitch EventSub topics. Quickly start up a webhooks endpoint and listen for Twitch EventSub notifications.

Visit TESjs:
NPM GitHub

Skip the Hard Stuff

TESjs makes it easy to get up and running by taking care of the tough stuff. Under the hood, TESjs uses Express to create a wehbooks endpoint for Twitch's EventSub notifications. It also takes care of the tedious signature verification so you can focus on building your awesome app!

Integrate Seamlessly

Already running an Express server in your app? TESjs has you covered! Just pass the Express object into TESjs' config and it will build a webhooks route on top of your existing server!

Install

 
npm install tesjs

Initialise the TES Instance

 
                            
const TES = require('tesjs');

// initialize TESjs
const tes = new TES({
    identity: {
        id: YOUR_CLIENT_ID,
        secret: YOUR_CLIENT_SECRET //do not ship this in plaintext!! use environment variables so this does not get exposed
    },
    listener: {
        baseURL: "https://example.com" //this MUST be an HTTPS endpoint on port 443
    }
});

Create an Event Listener

 
                            
// define an event handler for the 'channel.update' event
// NOTES: 
//   this handles ALL events of that type
//   events will not be fired until there is a subscription made for them
tes.on('channel.update', (userId, userName, title, language, categoryId, categoryName, isMature) => {
    console.log(`${userName}'s new title is ${title}`);
});

Subscribe to an Event

 
                            
// create a new subscription for the 'channel.update' event for broadcaster '1337'
tes.subscribe('channel.update', {
    broadcaster_user_id: '1337'
}).then(_ => {
    console.log('Subscription successful');
}).catch(err => {
    console.log(err);
});

Clean Up Revoked Subscriptions

 
                            
// the 'revocation' event will be fired if an subscription gets revoked by Twitch
tes.on('revocation', (subscriptionId, status, type, version, condition, transport, createdAt) => {
    console.log(`subscription with id ${subscriptionId} has been revoked`);
    // perform cleanup here
});
